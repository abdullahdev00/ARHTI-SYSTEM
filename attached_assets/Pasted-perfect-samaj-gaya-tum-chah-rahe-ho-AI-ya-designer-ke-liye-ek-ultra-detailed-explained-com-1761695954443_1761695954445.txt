## 🧾 Super Comprehensive Prompt (for UI Design Generation)

---

**Title:**
💼 **Arhti Business Management System — Comprehensive Responsive UI Design**

---

**Goal / Purpose:**
Design a **professional, responsive, and bilingual-ready** web and mobile interface for **Arhti (Mandi Middleman)** to manage his entire business workflow — including purchases from farmers, payments, commissions, invoices, and mandi charges — all in one clean dashboard system.

This design will be used later in a **React + Tailwind + shadcn + Framer Motion** frontend.
Now we only want **complete UI layout and design visuals**, not backend or logic.

---

### 🧩 1. Global Style Guide

**Design Framework & Libraries:**

* React + Vite base layout (for structure idea)
* TailwindCSS for styling system
* shadcn/ui components (Cards, Tables, Buttons, Dialogs, etc.)
* Framer Motion for smooth page transitions and hover animations

**Theme Modes:**

* Dual theme: **Light & Dark**
* User can switch manually OR follow system preference

**Typography:**

* Font: “Inter” or “Manrope”
* Font weights: 400 (normal), 500 (medium), 700 (bold)
* Headings should be clean, bold, minimal
* Subtext / labels use smaller muted tones (gray.500)

**Color Palette:**

```plaintext
Primary: #16a34a (green)
Accent: #facc15 (yellow)
Light Background: #ffffff
Dark Background: #0d1117
Text Light Mode: #1f2937
Text Dark Mode: #e5e7eb
Border Color: rgba(0,0,0,0.1)
```

**Shape & Shadows:**

* Corner Radius: 1.25rem (rounded-2xl)
* Card Shadow: soft-md, light blur
* Hover Effects: scale-up slightly with subtle shadow glow

**Spacing & Layout Grid:**

* Container width: 1200px max
* Padding inside cards: 1rem
* Gap between components: 1.5rem
* Consistent spacing across desktop/tablet/mobile

---

### 📱 2. Responsive Layout

#### 💻 Desktop Layout

* **Sidebar** navigation on left with collapsible feature
* Top navbar (contains user profile dropdown + theme toggle + search)
* Main content area scrollable
* Clean dashboard-style 3-column grid where possible

#### 📱 Mobile Layout

* Sidebar replaced with **bottom navigation bar** (rounded floating bar)
* Top bar only for page title + search
* All dialogs (forms, add new, filters) become **bottom sheets** instead of popups
* Mobile-first typography, minimal clutter

---

### 🧭 3. Navigation Structure

#### **Desktop Sidebar**

* Icons + labels (Lucide or HeroIcons style)

```
[Dashboard]
[Farmers]
[Purchases]
[Invoices]
[Payments]
[Charges]
[Reports]
[Settings]
```

* Collapsible sidebar animation with Framer Motion.
* Active item highlighted in green with subtle background.

#### **Mobile Bottom Navbar**

```
🏠 Dashboard
👥 Clients
📄 Invoices
💵 Payments
⋯ More
```

The “More” button opens a **bottom sheet** showing:

* Settings
* Theme switch
* Reports
* About / Help

---

### 📊 4. Page-by-Page Design Specification

---

#### 🏠 **Dashboard Page**

**Purpose:** Overview of Arhti’s entire activity.
**Components:**

1. **Header:** “Welcome back, [User Name]” with date & time
2. **Statistic Cards (4):**

   * Total Farmers
   * Pending Payments
   * Total Commission
   * Charges Today
     Each card has: icon + label + numeric value + small trend indicator (+/- %).
3. **Recent Activity Timeline:**

   * Example: “Received ₹10,000 from Factory A – 2h ago”
4. **Graph Section:**

   * Line or Bar chart showing monthly commission & purchases
5. **Quick Actions:**

   * Buttons: “Add Purchase”, “Create Invoice”, “Add Payment”
   * Buttons are colored (green for main, gray for secondary)

---

#### 👨‍🌾 **Farmers Page**

**Purpose:** Maintain list of all farmers Arhti deals with.
**UI Layout:**

1. Search bar + Filter dropdown (crop, date, status)
2. Toggle button: “Table View / Card View”
3. **Card View (mobile)**

   * Farmer name
   * Crop type
   * Last deal date
   * Total amount traded
   * Status badge (Active / Inactive)
4. **Table View (desktop)**
   Columns: Name | Crop | Weight | Rate | Total | Status | Date | Actions
5. “Add Farmer” button opens a dialog/bottom sheet with form fields:

   * Full Name, Phone, Address, Crop, Rate, Notes

---

#### 🌾 **Purchases Page**

**Purpose:** Add or view purchases from farmers.
**Layout:**

1. Header: “All Purchases” + “Add Purchase” button
2. Search bar
3. Table: Farmer | Crop | Quantity | Rate/kg | Total | Date | Payment Status
4. “Add Purchase” opens a **form dialog**:

   * Select Farmer (dropdown)
   * Crop Type
   * Quantity (kg)
   * Rate per kg
   * Date
   * Notes

---

#### 🧾 **Invoices Page**

**Purpose:** Manage all sale invoices.
**Layout:**

1. Header with “Create Invoice” button
2. Search + Filter (status: Paid/Unpaid)
3. Table/List: Invoice ID | Farmer | Date | Total | Status | View/Download
4. Invoice Detail Modal:

   * Breakdown of items
   * Commission deducted
   * Net payable amount
   * Option: “Mark as Paid”

---

#### 💰 **Payments Page**

**Purpose:** Track payments received from factories and paid to farmers.
**UI Layout:**

1. **Summary Cards (3):**

   * Received from Factory
   * Paid to Farmers
   * Pending Amount
2. **Payments Table:**
   Columns: Farmer | Invoice | Amount | Type | Status | Date
3. “Add Payment” button with dialog:

   * Type (Incoming/Outgoing)
   * Name
   * Amount
   * Date
   * Notes

---

#### ⚙️ **Charges Page**

**Purpose:** Record mandi or operational charges.
**Components:**

1. Button: “Add Charge”
2. Table: Charge Title | Amount | Type | Date | Notes
3. Dialog Form:

   * Charge Title
   * Amount
   * Type (Labor / Transport / Misc)
   * Applied To (Farmer / Purchase / Batch)
   * Notes

---

#### 📈 **Reports Page**

**Purpose:** View summary and analytics.
**Design:**

* Date range filter (calendar picker)
* Charts:

  * Pie chart (Charges by Type)
  * Line chart (Monthly Profit Trend)
  * Bar chart (Payments Comparison)
* Export buttons: PDF / Excel

---

#### ⚙️ **Settings Page**

**Purpose:** Personal & system preferences.
**Sections:**

1. **Theme Selection:**

   * System Default / Light / Dark
2. **Commission Settings:**

   * Commission Type: “Fixed %” or “Per Transaction”
   * Percentage input
3. **Currency & Date Format** dropdowns
4. Save button at bottom (fixed position on mobile)

---

### ⚡ 6. Animations & Microinteractions

* Page transitions: fade + slide-up (Framer Motion)
* Button hover: slight scale-up + glow border
* Dialogs & Sheets: smooth slide from bottom
* Sidebar: animated expand/collapse
* Table row hover: subtle highlight
* Charts animate-in when in viewport

---

### 🧱 7. Component Library Mapping (Shadcn/UI)

| Purpose | Component                                       |
| ------- | ----------------------------------------------- |
| Buttons | `<Button />` (variant: default, outline, ghost) |
| Cards   | `<Card />`                                      |
| Modals  | `<Dialog />`                                    |
| Tables  | `<Table />`, `<TableRow />`, `<TableCell />`    |
| Inputs  | `<Input />`, `<Select />`, `<Textarea />`       |
| Toggles | `<Switch />`, `<RadioGroup />`                  |
| Charts  | Custom Recharts components                      |
| Tabs    | `<Tabs />` for reports or grouped views         |

---

### 🧩 8. UX Enhancements

* Sticky header in tables
* Floating “Add” button on mobile bottom corner
* Search always visible on top
* Breadcrumbs on desktop (Dashboard / Farmers / Details)
* Confirmations before deletion
* Snackbar/toast for success/error alerts
* Auto theme sync via `window.matchMedia('(prefers-color-scheme: dark)')`

---

### ✅ 9. Design Goals Recap

* Simple enough for non-technical Arhti users
* Mobile-first
* Smooth visual experience
* Clear financial visibility
* Consistent UI system (colors, paddings, fonts, animations)
* Ready for next step: add authentication & backend later
